# История изменений

## [1.1.0] — 2026-03-20
- 🚀 Версия для разработки в ветке main

## [1.0.12] — 2026-03-20
- 🔒 Безопасность: устранён TOCTOU в `read_socket()` — `file_exists()` заменён атомарным `lstat()` + проверкой `is_link()`
- 🔒 Безопасность: `read_socket()` — добавлен лимит чтения (10 строк / 256 байт) для защиты от DoS через переполнение ответа из сокета
- 🔒 Безопасность: `stop_instance()` — `unlink()` теперь проверяет `is_link()` перед удалением pid/sock файлов
- 🔒 Безопасность: `stop_instance()` — `pkill -f` заменён завершением по точному PID для исключения случайного убийства чужого процесса
- 🔒 Безопасность: `start_instance()` — результат `pgrep` проверяется через `ps` по точному пути к сокету
- 🔒 Безопасность: права Unix-сокета ужесточены до `0o600` (только владелец)
- 🔒 Безопасность: SSRF — полная защита IPv6: заблокированы multicast (`ff00::/8`), link-local (`fe80::/10`), site-local (`fec0::/10`), unique-local (`fc00::/7`), IPv4-mapped адреса (`::ffff:127.x.x.x`), broadcast, зарезервированные диапазоны
- 🔒 Безопасность: SSRF — Python-зонд блокирует `is_reserved` и `is_multicast` через модуль `ipaddress`
- 🔒 Безопасность: защита от symlink attack — `unlink()` проверяет `is_link()` перед удалением socket/pid файлов
- 🔒 Безопасность: устранён TOCTOU — проверка существования сокета заменена атомарным `lstat()` + маска `S_IFSOCK`
- 🔒 Безопасность: добавлена эксклюзивная блокировка на `reconfigure` для защиты от race condition при параллельных вызовах
- 🔒 Безопасность: добавлена валидация диапазонов `probe_count` (1–20), `probe_interval` (5–300), `probe_timeout` (1–30) в PHP
- 🔒 Безопасность: добавлена валидация входных параметров в Python-зонде: `gw_name`, `probe_host`, `probe_port`, `probe_if`
- 🔒 Безопасность: `probe_port` проверяется как целое число 1–65535; `probe_if` — по шаблону `[a-zA-Z0-9_]+` в PHP
- 🔒 Безопасность: `probe_host` заблокирован для loopback (127.0.0.0/8, ::1), link-local (169.254.0.0/16), unspecified в PHP и Python
- 🔒 Безопасность: добавлена валидация `gw_name` против path traversal (только `[a-zA-Z0-9_-]`)
- 🔒 Безопасность: добавлена проверка формата UUID перед передачей в shell-команды
- 🔒 Безопасность: `shell_exec("kill -0 ...")` заменён на `posix_kill()`
- 🔒 Безопасность: добавлена блокировка файла при работе с `/tmp/gateways.status`
- 🔒 Безопасность: добавлен `htmlspecialchars()` для описаний интерфейсов в API
- 🔒 Безопасность: API больше не возвращает сырой вывод configd при ошибках
- 🐛 Исправлено некорректное имя лог-файла (`tun2socks_socket.log` → `gwmonitor_<name>.log`)

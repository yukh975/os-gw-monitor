# История изменений

## [1.0.11] — 2026-03-20
- 🔒 Безопасность: защита от symlink attack — `unlink()` теперь проверяет `is_link()` перед удалением socket/pid файлов
- 🔒 Безопасность: устранён TOCTOU — проверка существования сокета заменена атомарным `lstat()` + битовая маска типа файла (без следования symlink)
- 🔒 Безопасность: добавлена валидация диапазонов `probe_count` (1–20), `probe_interval` (5–300), `probe_timeout` (1–30) в PHP
- 🔒 Безопасность: API больше не возвращает сырой вывод configd при ошибках — детали заменены на общие сообщения

## [1.0.10] — 2026-03-20
- 🔒 Безопасность: защита от SSRF — заблокированы loopback (127.0.0.0/8, ::1), link-local (169.254.0.0/16, fe80::/10) и неопределённые адреса в Python-зонде и PHP-сервисе
- 🔒 Безопасность: добавлена валидация `probe_host` в PHP (loopback/link-local/unspecified заблокированы)
- 🔒 Безопасность: добавлена эксклюзивная блокировка на `reconfigure` для защиты от race condition при параллельных вызовах
- 🐛 Исправлено: расширен список блокируемых имён хостов (`localhost`, `ip6-localhost`, `ip6-loopback`)

## [1.0.9] — 2026-03-20
- 🔒 Безопасность: добавлена валидация входных параметров в Python-зонде (gw_name, probe_host, probe_port, probe_if)
- 🔒 Безопасность: probe_port проверяется как целое число в диапазоне 1–65535 в PHP и Python
- 🔒 Безопасность: probe_if проверяется по безопасному шаблону `[a-zA-Z0-9_]+` в PHP
- 🔒 Безопасность: устранено информационное раскрытие — ошибки backend больше не возвращают сырой вывод configd
- 🐛 Исправлено некорректное имя лог-файла (`tun2socks_socket.log` → `gwmonitor_<name>.log`)

## [1.0.8] — 2026-03-20
- 🔒 Безопасность: права Unix-сокета изменены с `0o666` на `0o660`
- 🔒 Безопасность: устранена race condition (TOCTOU) при создании сокета
- 🔒 Безопасность: добавлена валидация `gw_name` против path traversal (только `[a-zA-Z0-9_-]`)
- 🔒 Безопасность: добавлена проверка формата UUID перед передачей в shell-команды
- 🔒 Безопасность: `shell_exec("kill -0 ...")` заменён на `posix_kill()`
- 🔒 Безопасность: добавлена блокировка файла при работе с `/tmp/gateways.status`
- 🔒 Безопасность: добавлен `htmlspecialchars()` для описаний интерфейсов в API

# История изменений

## [1.1.0] — 2026-03-20

### Безопасность
- 🔒 `gw_monitor_probe.py` — добавлен семафор подключений (`_MAX_CONN = 10`) для защиты от DoS через бесконтрольное порождение потоков при accept
- 🔒 `gw_monitor_probe.py` — таймаут соединения (`_CONN_TIMEOUT = 5с`) исключает блокировку всех слотов семафора зависшими клиентами
- 🔒 `gw_monitor_probe.py` — лог-файл открывается с `O_NOFOLLOW` атомарно, исключая TOCTOU-гонку с симлинком
- 🔒 `gw_monitor_probe.py` — защита от symlink-атаки на лог-файл: процесс завершается, если путь лога является симлинком
- 🔒 `gw_monitor_probe.py` — Python-процесс записывает свой PID атомарно при старте; исключает ненадёжный поиск через `pgrep`/`ps` в PHP
- 🔒 `gw_monitor_probe.py` — PID-файл создаётся с правами `0o600` (только владелец)
- 🔒 `gw_monitor_probe.py` — `sock_path` (argv[1]) валидируется по шаблону `/var/run/dpinger_<name>.sock`
- 🔒 `gw_monitor_probe.py` — добавлена валидация диапазонов `count` (1–20), `interval` (5–300), `timeout` (1–30) с try/except на `int()`
- 🔒 `gw_monitor_probe.py` — IPv6-адреса без скобок автоматически оборачиваются в `[]` в URL для curl
- 🔒 `gwmonitor-service.php` — `exec("kill")` заменён на `posix_kill(SIGTERM/SIGKILL)` — безопасное завершение без fork/shell
- 🔒 `gwmonitor-service.php` — защита от DNS rebinding: хостнейм резолвится один раз в момент валидации; полученный IP проверяется против запрещённых диапазонов
- 🔒 `gwmonitor-service.php` — добавлен `stream_set_timeout(1)` на чтение из сокета для исключения бесконечной блокировки в `read_socket()`
- 🔒 `gwmonitor-service.php` — добавлена проверка `is_numeric()` на данные из сокета перед приведением к int
- 🔒 `gwmonitor-service.php` — добавлена проверка `is_link()` перед `fopen()` lock-файла reconfigure
- 🔒 `gwmonitor-service.php` — lock-файл reconfigure удаляется после использования (`@unlink`)
- 🔒 `gwmonitor-cleanup.php` — добавлено ограничение размера 1 МБ перед `unserialize()` для защиты от переполнения памяти через подменённый `gateways.status`
- 🔒 `gwmonitor-cleanup.php` — проверка возврата `simplexml_load_file()`; корректный выход при ошибке парсинга
- 🔒 `gwmonitor-list-interfaces.php` — имена интерфейсов из `ifconfig` валидируются по `[a-zA-Z0-9_-]+`
- 🔒 `MonitorController.php` — пустой `gw_name` теперь возвращает ошибку валидации в `addMonitor` и `setMonitor` (ранее обходил проверку дублей и падал в сохранение модели)
- 🔒 `Monitor.xml` — маска `probe_host` расширена для приёма IPv6-адресов в скобках (`[2001:db8::1]`), приведена в соответствие с PHP и Python слоями
- 🔒 `install.sh` — `pkill -f` заменён на `_kill_monitors()` с убийством по PID-файлам во избежание случайного завершения чужих процессов
- 🔒 `install.sh` — `_kill_monitors()` теперь проверяет через `ps`, что PID принадлежит `gw_monitor_probe`, перед отправкой сигнала (защита от переиспользованных PID)
- 🔒 `install.sh` — все `rm -f /tmp/opnsense_menu_cache.xml` теперь защищены проверкой `[ ! -L ... ]` для предотвращения удаления файлов через symlink
- 🔒 `gw_monitor.inc` — null-результат `shell_exec()` теперь обнаруживается и выводится как ошибка

### Новые возможности
- ✨ `install.sh` — добавлена команда `reinstall`: тихо удаляет текущую версию (с сохранением настроек) и устанавливает заново, обходя проверку «уже установлено»

### Качество кода
- 🔧 `gwmonitor-cleanup.php` — `goto` заменён структурным блоком `if/else`

## [1.0.12] — 2026-03-20

### Безопасность
- 🔒 Устранён TOCTOU в `read_socket()` — `file_exists()` заменён атомарным `lstat()` + проверкой `is_link()`
- 🔒 `read_socket()` — добавлен лимит чтения (10 строк / 256 байт) для защиты от DoS через переполнение ответа из сокета
- 🔒 `stop_instance()` — `unlink()` теперь проверяет `is_link()` перед удалением pid/sock файлов
- 🔒 `stop_instance()` — `pkill -f` заменён завершением по точному PID для исключения случайного убийства чужого процесса
- 🔒 `start_instance()` — результат `pgrep` проверяется через `ps` по точному пути к сокету
- 🔒 Права Unix-сокета ужесточены до `0o600` (только владелец)
- 🔒 SSRF — полная защита IPv6: заблокированы multicast (`ff00::/8`), link-local (`fe80::/10`), site-local (`fec0::/10`), unique-local (`fc00::/7`), IPv4-mapped адреса (`::ffff:127.x.x.x`), broadcast, зарезервированные диапазоны
- 🔒 SSRF — Python-зонд блокирует `is_reserved` и `is_multicast` через модуль `ipaddress`
- 🔒 Защита от symlink attack — `unlink()` проверяет `is_link()` перед удалением socket/pid файлов
- 🔒 Устранён TOCTOU — проверка существования сокета заменена атомарным `lstat()` + маска `S_IFSOCK`
- 🔒 Добавлена эксклюзивная блокировка на `reconfigure` для защиты от race condition при параллельных вызовах
- 🔒 Добавлена валидация диапазонов `probe_count` (1–20), `probe_interval` (5–300), `probe_timeout` (1–30) в PHP
- 🔒 Добавлена валидация входных параметров в Python-зонде: `gw_name`, `probe_host`, `probe_port`, `probe_if`
- 🔒 `probe_port` проверяется как целое число 1–65535; `probe_if` — по шаблону `[a-zA-Z0-9_]+` в PHP
- 🔒 `probe_host` заблокирован для loopback (127.0.0.0/8, ::1), link-local (169.254.0.0/16), unspecified в PHP и Python
- 🔒 Добавлена валидация `gw_name` против path traversal (только `[a-zA-Z0-9_-]`)
- 🔒 Добавлена проверка формата UUID перед передачей в shell-команды
- 🔒 `shell_exec("kill -0 ...")` заменён на `posix_kill()`
- 🔒 Добавлена блокировка файла при работе с `/tmp/gateways.status`
- 🔒 Добавлен `htmlspecialchars()` для описаний интерфейсов в API
- 🔒 API больше не возвращает сырой вывод configd при ошибках

### Исправления
- 🐛 Исправлено некорректное имя лог-файла (`tun2socks_socket.log` → `gwmonitor_<name>.log`)

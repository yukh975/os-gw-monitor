# История изменений

Все значимые изменения os-gw-monitor задокументированы здесь.

## [1.0.7] — 2026-03-19

### Исправлено
- Файл версии `/var/db/gwmonitor-version` не удалялся при удалении плагина, из-за чего при повторной установке скрипт ошибочно считал плагин уже установленным

## [1.0.6] — 2026-03-19

- `README_RU.md` — документация на русском языке с перекрёстными ссылками на английскую версию
- Бейджи (версия, платформа, лицензия) в обоих README файлах

### Изменено
- `README.md` переведён на английский как основной файл документации
- Файл версии перенесён из `/usr/local/sbin/gwmonitor-version` в `/var/db/gwmonitor-version`
- Добавлено имя автора в README файлы

### Удалено
- `plugin/usr/local/sbin/gwmonitor-version` — заменён на `plugin/var/db/gwmonitor-version`

## [1.0.5] — 2026-03-18

## [1.0.4] — 2026-03-18

### Исправлено

## [1.0.3] — 2026-03-18

### Исправлено
- Подавлен лишний вывод от `pkill`, `killall` и `pluginctl -c monitor` при установке и удалении

---

## [1.0.2] — 2026-03-18

### Добавлено
- Проверка версии при установке: если та же версия уже установлена — установка пропускается
- Автообновление: если обнаружена другая версия — выполняется тихое удаление (с сохранением настроек) перед установкой новой
- Функция `do_uninstall_silent` для процесса обновления
- Версия сохраняется в `/var/db/gwmonitor-version`
- Версия отображается на странице мониторинга в GUI

---

## [1.0.1] — 2026-03-18

### Добавлено
- Первый релиз плагина
- Мониторинг шлюзов через HTTP с поддержкой нескольких инстансов
- GUI по адресу `System → Gateways → Monitoring` (MVC-плагин с таблицей UIBootgrid)
- Настройки каждого инстанса: имя шлюза, интерфейс, хост зондирования, порт, количество проб, интервал, таймаут, описание
- Выпадающие списки шлюзов и интерфейсов из конфигурации OPNsense
- Защита от дублирования шлюзов в `MonitorController`
- `gw_monitor_probe.py` — Python-демон: HTTP-зондирование + Unix-сокет сервер в формате dpinger
- `gwmonitor-service.php` — бэкенд: читает config.xml, запускает и останавливает инстансы
- `gwmonitor-list-interfaces.php` — возвращает список физических интерфейсов для выпадающих списков GUI
- `gwmonitor-cleanup.php` — очищает сокеты, pid-файлы и кеш статусов шлюзов при удалении
- `gw_monitor.inc` — plugin hook OPNsense: регистрирует сервисы и хук `monitor` для автоперезапуска после изменения настроек шлюзов
- `actions_gwmonitor.conf` — действия configd: reconfigure, status, start, stop, watchdog, listinterfaces
- `install.sh` с режимами install и uninstall
- При удалении: выбор — сохранить или удалить настройки из config.xml
- При удалении: очистка кеша статусов шлюзов, удаление сокетов, восстановление штатного dpinger
- Очистка кеша меню при установке и удалении
- Второй перезапуск `configd` в конце установки для гарантированной загрузки всех actions
- Поддержка watchdog через OPNsense Cron (`Gateway Monitor Watchdog`)
